## Ручной сценарий приемки

Перед сдачей вертикального среза пройти вручную:

1. Запустить сцену `Level_01`.
2. Проверить перемещение игрока.
3. Создать первое эхо.
4. Создать второе эхо.
5. Переключиться между эхо и проверить визуальную индикацию.
6. Выполнить swap с первым эхо.
7. Убедиться, что эхо переместилось на старую позицию игрока.
8. Выполнить повторный swap после cooldown.
9. Проверить реакцию на попытку создать лишнее эхо.
10. Проверить, что без эхо уровень не проходится.
11. Пройти уровень с использованием кнопок, дверей и опасных зон.
12. Достичь выхода и завершить уровень.

## Риски

| Риск | Как снижать |
| --- | --- |
| Игрок или эхо появляются внутри геометрии после swap | Делать capsule/overlap-проверки перед обменом |
| Список эхо рассинхронизируется | Все создание, удаление и выбор проводить через `EchoManager` |
| Активное эхо непонятно игроку | Использовать явное визуальное выделение и HUD |
| Уровень проходится без эхо | Проверять puzzle-дизайн вручную и через приемочный сценарий |
| Рестарт ломает состояние объектов | Все объекты уровня подключать к `IResettableLevelObject` |
| UI начинает управлять gameplay | UI только подписывается на события и вызывает явные команды |
